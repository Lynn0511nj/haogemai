# 长图报告内容更新说明

## 更新内容

长图报告现在展示完整诊断报告的核心信息，包括：

### 1. 账号定性
- 显示完整诊断报告的核心状态 (`fullReport.coreStatus`)
- 如果没有完整报告，则显示快速诊断的阶段 (`clarityPhase`)
- 使用更紧凑的字体大小（text-2xl）以适应更多内容

### 2. 核心问题
- 新增独立的"核心问题"板块
- 显示 `fullReport.coreProblem`
- 使用米色背景和橙色左边框突出显示

### 3. 下一步行动（可实操示例）
- 完整展示 `fullReport.actionableExample`
- 包含三个部分：
  - 背景说明 (context)
  - 具体示例 (example) - 浅绿色背景
  - 执行限制 (constraint) - 浅黄色背景
- 用户可以直接参考这些具体的行动建议

### 4. 7天优化行动清单
- 优化了时间线布局
- 每个行动项包含：
  - 时间标签 (day)
  - 具体行动 (action)
  - 预期目标 (goal)
- 使用时间线样式，更清晰地展示行动步骤

### 5. 底部信息
- 优化了二维码大小（从 20x20 改为 16x16）
- 调整了文字大小以适应更多内容

## 设计优化

1. **信息层次更清晰**：使用不同的颜色标识符区分各个板块
2. **内容更实用**：展示用户最需要的核心问题和行动建议
3. **布局更紧凑**：优化间距和字体大小，在有限空间内展示更多信息
4. **视觉更统一**：与完整诊断报告页面的设计风格保持一致

## 技术实现

- 使用条件渲染 (`result.fullReport?.actionableExample &&`) 确保数据存在时才显示
- 保持响应式设计，适配 375px 宽度的长图
- 使用 html2canvas 生成高清图片（scale: 2）
