# Prompt B 更新说明

## 更新目标

将 Prompt B 从"深度分析"改为"可执行陪跑方案"，实现：
- 🎯 给方法、给例子、给清单
- 💰 让人觉得钱花对了
- 🚀 每个结论都能转化为行动
- 🎓 不追求全面，只解决最关键问题

## 核心变化

### 1. 角色定位
**旧版**: AI 诊断顾问，提供深度分析  
**新版**: 深度陪跑诊断师，输出可执行方案

### 2. 核心原则
- ✅ 不追求全面，只解决最关键问题
- ✅ 每个结论都要能转化为行动
- ✅ 不输出空洞方法论
- ✅ 假设用户是迷茫期，但愿意行动

### 3. 输出结构（6部分）

#### 第一部分｜一句话总判断
- 用一句话概括账号当前状态
- **限制**: 不超过 20 字
- **示例**: "内容有价值但缺少稳定标签"

#### 第二部分｜核心矛盾拆解（只选一个）
- 明确指出：当前最优先解决的问题是什么
- 说明：为什么不先解决其他问题
- **限制**: 100字内
- **示例**: "你目前最优先解决的是「账号标签混乱」，而不是内容质量。因为算法需要先判断你是谁，才能决定推给谁。"

#### 第三部分｜对标案例启发（新增）
- 描述 1 种同类型账号的常见做法
- 重点放在「怎么改」，不是「谁」
- **限制**: 80字内
- **示例**: "同类型账号通常会在简介中明确「身份+领域+价值」，然后前10条内容只讲一个方向，让算法快速打标签。"

#### 第四部分｜下一步可执行操作（重点）
给出 3 条具体、低门槛、可马上执行的调整建议：
1. **今天可以做的**: 修改简介，明确身份标签
2. **本周可以完成的**: 整理主页前10条内容
3. **容易被忽略但很关键的点**: 不要在简介里写泛词

#### 第五部分｜7 天优化行动清单
- **Day 1 – Day 7**: 每天独立任务
- **格式**: 一句话任务 + 预期结果
- **原则**: 不解释原理，只给行动

#### 第六部分｜留白与陪跑引导（不推销）
- 说明哪些问题适合在 1 对 1 中结合具体情况拆解
- 强调陪跑的价值是「持续校准，而不是一次建议」
- **限制**: 80字内

## 输出格式变化

### 旧版输出
```json
{
  "coreStatus": "强断言式定性",
  "coreProblem": "核心矛盾诊断",
  "actionableExample": {
    "title": "下一步，你可以这样试一次",
    "context": "背景说明",
    "example": "可直接模仿的示例",
    "constraint": "执行限制"
  },
  "sevenDayPlan": [
    { "day": "Day 1-2", "action": "...", "goal": "..." }
  ]
}
```

### 新版输出
```json
{
  "coreStatus": "一句话总判断（≤20字）",
  "coreProblem": "核心矛盾拆解（≤100字）",
  "benchmarkInsight": "对标案例启发（≤80字）",
  "actionableExample": {
    "title": "下一步可执行操作",
    "context": "今天可以做的",
    "example": "本周可以完成的",
    "constraint": "容易被忽略但很关键的点"
  },
  "sevenDayPlan": [
    { "day": "Day 1", "action": "一句话任务", "goal": "预期结果" },
    { "day": "Day 2", "action": "一句话任务", "goal": "预期结果" },
    // ... Day 3-7
  ],
  "closingMessage": "留白与陪跑引导（≤80字）"
}
```

## UI 变化

### 新增元素
1. **对标案例启发**
   - 蓝色背景卡片
   - 左侧蓝色边框
   - 展示同类型账号的常见做法

### 优化元素
1. **7天行动清单**
   - 从 3 个时间段改为 7 天独立任务
   - 每天一个具体行动 + 预期结果
   - 更清晰的时间线布局

2. **下一步可执行操作**
   - 重新定义三个部分的含义
   - 更强调"可马上执行"

## 语气变化

### 旧版
- 像经验型顾问
- 温和、笃定
- 强调"被看懂感"

### 新版
- 冷静、有经验
- 像一个在旁边看你实操的人
- 不夸张、不制造焦虑

## 价值感提升

### 1. 更具体的建议
**旧版**: "优化账号定位"  
**新版**: "修改简介，明确「你是谁+做什么+给什么价值」，控制在15字内"

### 2. 更低的门槛
**旧版**: "重新规划内容方向"  
**新版**: "今天：修改简介。本周：整理主页前10条内容"

### 3. 更清晰的行动路径
**旧版**: Day 1-2 做什么  
**新版**: Day 1 做什么，Day 2 做什么，Day 3 做什么...

### 4. 新增对标案例
- 让用户看到"别人是怎么做的"
- 降低执行难度
- 增加可信度

## 示例对比

### 旧版示例
```
coreStatus: "你的账号目前处在「内容不差，但还没被算法当成一个人」的阶段"
coreProblem: "你的内容横跨多个方向，导致算法无法判断该把你推给哪一类人"
actionableExample: {
  context: "你现在不需要重新规划定位，而是先做一次身份收敛测试",
  example: "发一条内容，只讲你在某领域的一个观点",
  constraint: "不要讲方法论，只讲你的个人视角"
}
```

### 新版示例
```
coreStatus: "内容有价值但缺少稳定标签"
coreProblem: "你目前最优先解决的是「账号标签混乱」，而不是内容质量。因为算法需要先判断你是谁，才能决定推给谁。内容再好，标签不清晰，流量就不精准。"
benchmarkInsight: "同类型账号通常会在简介中明确「身份+领域+价值」，比如「3年运营 | 教你用数据做决策」。然后前10条内容只讲一个方向，让算法快速打标签。"
actionableExample: {
  context: "今天：修改简介，明确「你是谁+做什么+给什么价值」，控制在15字内",
  example: "本周：把最近5条内容的标题统一改成同一个方向的表达，让主页看起来是「专注做一件事的人」",
  constraint: "容易被忽略的点：不要在简介里写「热爱生活」「记录日常」这类泛词，算法无法识别"
}
```

## 转化策略优化

### 旧版
- 强调"诊断 ≠ 改变"
- 制造"现在不动会继续卡住"的感觉
- 较强的推销感

### 新版
- 强调"持续校准，而不是一次建议"
- 说明哪些问题适合在 1 对 1 中拆解
- 不推销，只说明价值

## 测试要点

1. **可执行性测试**
   - 验证每个建议都能立即执行
   - 确认门槛足够低
   - 检查是否有空洞方法论

2. **价值感测试**
   - 用户是否觉得"钱花对了"
   - 是否有"这个我能做"的感觉
   - 是否有"原来可以这样"的启发

3. **完整性测试**
   - 7天清单是否覆盖完整
   - 每天的任务是否独立
   - 预期结果是否明确

4. **转化测试**
   - 对标案例是否有启发
   - 留白引导是否自然
   - 是否有继续咨询的欲望

## 关键指标

### 用户满意度
- 目标：让用户觉得"钱花对了"
- 衡量：是否立即开始执行建议

### 可执行性
- 目标：每个建议都能转化为行动
- 衡量：用户是否能理解并执行

### 转化率
- 目标：自然引导到 1 对 1 陪跑
- 衡量：添加微信的比例
