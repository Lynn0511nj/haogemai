# 诊断进行中提示语设计

## 功能说明

在用户提交诊断请求后，后台分析过程中会轮播显示 3-5 条提示语，用于安抚用户等待情绪。

## 提示语列表

当前使用的 5 条提示语：

1. **"正在扫描你的账号结构…"**
   - 传达：开始分析阶段
   - 确定感：正在扫描（进行时）

2. **"已匹配同类账号成长模型…"**
   - 传达：数据对比阶段
   - 确定感：已匹配（完成时）

3. **"分析你的主页信号强度…"**
   - 传达：深度分析阶段
   - 确定感：分析（进行时）

4. **"对比迷茫期常见卡点中…"**
   - 传达：问题定位阶段
   - 确定感：对比中（进行时）

5. **"正在生成你的关键判断…"**
   - 传达：结论生成阶段
   - 确定感：正在生成（进行时）

## 设计原则

### ✅ 要求

1. **字数限制**：每条不超过 20 字
2. **确定感**：偏"正在分析 / 已为你扫描"的确定感
3. **禁止模糊词**：不出现"可能 / 也许"等不确定词汇
4. **进度感**：体现分析的不同阶段
5. **专业感**：使用专业术语（如：信号强度、成长模型、卡点）

### ❌ 避免

- 使用"可能"、"也许"、"大概"等模糊词
- 超过 20 字的长句
- 过于技术化的术语
- 重复的表达方式

## 技术实现

### 轮播机制

```typescript
const analyzingTips = [
  "正在扫描你的账号结构…",
  "已匹配同类账号成长模型…",
  "分析你的主页信号强度…",
  "对比迷茫期常见卡点中…",
  "正在生成你的关键判断…"
];
```

### 切换逻辑

- **切换间隔**：2.5 秒
- **过渡时间**：300ms 淡入淡出
- **循环方式**：顺序循环播放

### 动画效果

1. **淡出**：当前提示语 opacity 从 1 → 0（300ms）
2. **切换**：更新提示语内容
3. **淡入**：新提示语 opacity 从 0 → 1（300ms）

## 用户体验

### 心理效果

1. **降低焦虑**：让用户知道系统正在工作
2. **建立信任**：展示专业的分析流程
3. **时间感知**：通过阶段性提示，让等待时间感觉更短
4. **确定性**：使用确定的动词，增强可信度

### 视觉层次

```
┌─────────────────────────────┐
│     [旋转的加载动画]          │
│                             │
│  正在扫描你的账号结构…       │  ← 主提示语（轮播）
│                             │
│ 不看冷冰冰的数据，           │  ← 副标题（固定）
│ 只看你表达的灵魂。           │
└─────────────────────────────┘
```

## 扩展建议

### 可选的额外提示语

如果需要增加更多提示语，可以考虑：

- "识别你的内容风格特征…"
- "定位你的目标受众画像…"
- "评估你的表达清晰度…"
- "检测你的信任感指数…"
- "计算你的流量健康度…"

### 动态提示语

未来可以根据用户输入的 `stressPoint` 动态生成提示语：

```typescript
const getDynamicTips = (stressPoint: string) => {
  const baseTips = [
    "正在扫描你的账号结构…",
    "已匹配同类账号成长模型…"
  ];
  
  const specificTips = {
    "没流量": "分析你的流量卡点原因…",
    "涨粉难/互动少": "评估你的互动吸引力…",
    "灵感匮乏": "扫描你的选题方向…",
    "定位太乱": "诊断你的定位清晰度…",
    "不懂变现": "匹配你的变现路径…"
  };
  
  return [...baseTips, specificTips[stressPoint], ...];
};
```

## 测试建议

1. **时长测试**：确保提示语轮播至少覆盖 10-15 秒的分析时间
2. **可读性测试**：确保用户能在 2.5 秒内读完每条提示语
3. **情感测试**：确认提示语能有效缓解用户焦虑
4. **一致性测试**：确保提示语风格与品牌调性一致
