# Prompt A 更新说明

## 更新目标

将 Prompt A 从"全面诊断"改为"极速初判"，实现：
- ⚡ 10秒内完成
- 📝 200字以内
- 🎯 快速下判断，不写长文
- 🔍 只判断一个核心问题

## 核心变化

### 1. 角色定位
**旧版**: AI 诊断顾问，提供全面分析  
**新版**: 极速诊断助手，快速下判断

### 2. 输出约束
- ❌ 不进行深度内容分析
- ❌ 不生成长段文字
- ❌ 不讨论变现、私域、长期策略
- ✅ 只判断一个「最核心问题」
- ✅ 输出必须在 200 字以内

### 3. 诊断流程（4步）

#### Step 1：判断账号所处的【实际状态】
从以下类型中选择一个：
- 新手探索期
- 内容已产出但方向不稳
- 有流量但难以沉淀
- 明显遇到增长瓶颈
- 记录型账号（无明确目标）

#### Step 2：给出一个【断言式结论】
格式：`"你目前最大的卡点不是【A】，而是【B】。"`  
⚠️ 必须是单一结论，不允许并列

#### Step 3：给出【1–2 条判断依据】
- 使用「因为……所以……」句式
- 不使用专业术语
- 基于用户选择 + 主页直观信息

#### Step 4：留一个【未展开的悬念】
说明还有哪些更深层维度已被检测，但暂不展示

```json
  "primaryImprovement": {
    "title": "立即可改进的方向",
    "whyItMatters": "为什么这样改",
    "actionableStep": "具体示例",
    "actionableExample": "执行提示"
  }
}
```

### 新版输出
```json
{
  "clarityPhase": "当前阶段判断（15字内）",
  "phaseInspiration": "核心卡点结论（断言式）",

  "primaryImprovement": {

    "whyItMatters": "维度1：信任感",
    "actionableStep": "维度2：内容记忆点",

  "dimensions": [

      "description": "简短判断（10字内）"
    }
    // ... 5个维度
  ]
}
```

## UI 变化

### 新增元素
1. **标题区域**
   - "你的账号初步判断"
   - "10秒极速诊断"

2. **当前阶段判断**
   - 渐变卡片展示
   - 简洁的阶段描述

3. **核心卡点结论**
   - 米色背景 + 橙色左边框
   - 断言式结论

4. **判断依据**
   - 独立区块
   - 清晰的依据说明

5. **未展开的诊断维度**
   - 虚线边框卡片
   - 锁图标
   - 列表展示3个维度

### 移除元素
- 旧版的"实操案例"区块
- 长段的共情文字

## 语气变化

### 旧版
- 温和、共情
- 详细解释
- 鼓励性语言

### 新版
- 笃定
- 克制
- 不安抚、不推销
- 像一个已经看过很多账号的人

## 示例对比

### 旧版示例
```
clarityPhase: "一辆动力十足但没有方向盘的赛车"
phaseInspiration: "你的内容质量不错，但缺少明确的定位"
empathyMessage: "我理解你的焦虑，很多创作者都会经历这个阶段..."
```

### 新版示例
```
clarityPhase: "内容已产出但方向不稳"
phaseInspiration: "你目前最大的卡点不是内容质量，而是账号标签混乱。"
empathyMessage: "因为你的简介和内容跨越了职场、生活、情感三个方向，所以算法无法判断该把你推给谁。"
```

## 性能优化

### 速度提升
- 减少输出长度（200字以内）
- 简化分析逻辑
- 目标：10秒内完成

### Token 优化
- 旧版：约 1500-2000 tokens
- 新版：约 500-800 tokens
- 节省：60-70%

## 转化策略

### 悬念设置
通过"未展开的诊断维度"制造悬念：
- 信任感建立路径
- 内容记忆点设计
- 变现承接能力

### 付费引导
明确告知用户：
- 这是免费的极速诊断
- 完整报告包含更深层分析
- 7天可执行清单
- 案例实操

## 测试要点

1. **速度测试**
   - 确认10秒内返回结果
   - 测试不同网络环境

2. **内容测试**
   - 验证输出在200字以内
   - 确认断言式结论清晰
   - 检查判断依据合理

3. **UI测试**
   - 验证新布局正确显示
   - 确认所有字段都有数据
   - 测试响应式布局

4. **转化测试**
   - 验证"未展开维度"吸引力
   - 测试付费按钮点击率
   - 确认付费流程顺畅
